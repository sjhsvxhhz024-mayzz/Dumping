# Что осталось сделать, чтобы дамп заработал — по шагам, для тупого

Цель: получить `dump.cs` — текстовый файл со всеми классами, их полями
и методами (как в исходниках C#), из защищённой игры.

Что УЖЕ работает (не трогаем, это готово):
- Расшифровка `global-metadata.dat` — файл `tools/decrypt_metadata.py`. Запустил -> получил расшифрованный файл.
- Мы знаем ключ (0xA5C3F19D) и что шифруется только начало файла (заголовок).
- Мы знаем, где внутри файла лежат имена классов и методов, и как они устроены.

Ниже — что ещё НЕ доделано. Идём строго по порядку. Каждый шаг = одна задача.

---

## ШАГ 1. Связать методы с классами (найти "methodStart")

Простыми словами:
У нас есть список из 29366 классов и отдельно список из 225735 методов.
Сейчас они лежат двумя кучами и НЕ связаны: непонятно, какой метод какому классу принадлежит.

В каждом классе внутри записан номер — "с какого метода в общей куче начинаются мои методы"
(называется `methodStart`) и "сколько их" (`method_count`). Надо найти, на каком месте
внутри записи класса лежат эти два числа.

Как понять что нашли правильно:
- `methodStart` у классов идёт по возрастанию (0, 5, 12, 20, ...) — никогда не прыгает назад.
- Если сложить methodStart последнего класса + его method_count = должно получиться 225735 (всего методов).

Результат шага: для каждого класса знаем его список методов.

---

## ШАГ 2. Связать поля с классами (найти "fieldStart")

То же самое, что ШАГ 1, но для полей класса (переменных), а не методов.
Ищем внутри записи класса числа `fieldStart` и `field_count` тем же способом
(идут по возрастанию, сумма сходится с общим числом полей).

Результат шага: для каждого класса знаем его список полей.

---

## ШАГ 3. Достать типы полей и возвращаемые типы методов

Простыми словами:
Мы уже умеем читать ИМЕНА (класс "Player", метод "Jump"). Но не знаем ТИПЫ:
`int`, `float`, `string`, `bool`, другой класс и т.д.

Типы лежат не в метадате, а рядом — их надо взять из библиотеки `libil2cpp.so`
(таблица типов). Нужно найти таблицу типов и научиться переводить "номер типа" -> "int/float/...".

Результат шага: у каждого поля и метода есть человекочитаемый тип.

---

## ШАГ 4. Вписать всё это в форк дампера (myfork.zip)

Простыми словами:
Готовый дампер `il2cpp-dumper-rs` умеет писать красивый `dump.cs`, НО он читает
файл по СТАРОМУ стандарту и на нашем ломается. Надо подменить в нём три куска чтения:

1. Чтение заголовка — под наш формат (31 "тройка", свой порядок). Данные уже есть в `tools/v39_header_layout.md`.
2. Чтение записи класса — размер 82 байта, imя на позиции 0, namespace на позиции 4,
   плюс methodStart/fieldStart из ШАГ 1-2.
3. Чтение записи метода — размер 32 байта, имя на позиции 0.

Файлы, которые правим (лежат в myfork.zip):
- `src/il2cpp/structures.rs` — тут описания записей (размеры и позиции полей).
- `src/il2cpp/metadata.rs` — тут порядок чтения заголовка.

Результат шага: `cargo build --release`, запускаем — дампер не падает.

---

## ШАГ 5. Проверить результат

Запуск:
```
./target/release/il2cpp_dumper \
  --binary libil2cpp.so \
  --metadata global-metadata.dat(расшифрованный) \
  --output ./out
```

Что проверяем в `out/dump.cs`:
- Классы с нормальными именами (например `Joystick`, `RockingBoat`) — есть.
- Внутри классов — методы (`Update`, `Start`, `.ctor`) и поля.
- Типы полей (`int`, `float`, `Vector3`) проставлены.
- Нет мусора вроде кракозябр в именах (обфусцированные короткие имена типа `WiJ` — это норма, не ошибка).

Если всё так — инструмент готов и постоянный: на следующей версии игры просто
повторяешь ШАГ "расшифровать" + запуск дампера.

## Запасной вариант (если статика забуксует на ШАГ 3)

Frida-дамп из памяти (файл-инструкции сделаю отдельно):
- нужен телефон с root или эмулятор (LDPlayer/Genymotion с root)
- ставится frida-server на устройство, `pip install frida-tools` на ПК
- запускаешь игру со скриптом — она сама расшифровывает всё в памяти, а мы читаем готовое
- выдаёт такой же `dump.cs` за ~15 минут, но нужен запущенный процесс игры

Разница: статика (ШАГ 1-5) = работает без телефона, из файлов.
Frida = быстрее и надёжнее, но нужен телефон/эмулятор с игрой.
